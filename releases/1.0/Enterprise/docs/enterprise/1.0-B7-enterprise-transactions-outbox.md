## Purpose

This release introduces the Enterprise Transaction Boundary and Outbox Foundation for Kenswell One Enterprise 1.0.

## Architecture Position

Enterprise Transactions sit after:

1. Enterprise Context
2. Enterprise Audit
3. Enterprise Workflow
4. Unit of Work

They provide the foundation for:

1. Commit and rollback boundaries
2. Reliable outbox events
3. Domain event dispatch after commit
4. Future persistence adapters
5. Future distributed saga workflows

## Transaction Lifecycle

The transaction lifecycle is:

1. Begin transaction boundary
2. Stage domain work
3. Stage outbox events
4. Commit transaction
5. Persist staged outbox events
6. Dispatch events asynchronously later

If rollback occurs, staged outbox events are discarded.

## Outbox Model

The outbox model prevents events from being dispatched before the business operation commits.

Current statuses:

- `pending`
- `dispatched`
- `failed`

Current event types:

- `domain.event`
- `audit.event`
- `notification.event`
- `integration.event`
- `workflow.event`
