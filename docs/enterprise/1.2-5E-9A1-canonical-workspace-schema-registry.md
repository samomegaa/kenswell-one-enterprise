# Release 1.2 — 5E-9A1

## Canonical Payroll Workspace Schema Registry

This release introduces the metadata-driven section registry for the Canonical Payroll Workspace.

The workspace is a presentation layer over the Staffology-native employee contract. It does not replace, reduce or reshape the underlying payroll payload.

## Canonical sections

1. Personal
2. Address
3. Employment
4. Tax & NI
5. Pay
6. Banking
7. Pension
8. Benefits & Deductions
9. Leave & Statutory Pay
10. Opening Balances
11. RTI
12. Provider

Each section defines a stable identifier, title, order, icon key, description, readiness contribution and optional visibility metadata.

## Design rules

- Workspace metadata is immutable.
- Section identifiers are stable API values.
- Duplicate registration is rejected.
- Provider information is isolated in its own section.
- The Staffology-native contract remains authoritative.
- Later releases add field metadata and dynamic rendering without changing this section registry.
