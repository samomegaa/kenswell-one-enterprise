# Release 1.2 — 5E-9A4

## Canonical Payroll Conditional Visibility Engine

This release introduces metadata-driven field and section visibility for the
Canonical Payroll Workspace.

## Supported operators

- `equals`
- `not-equals`
- `truthy`
- `falsy`
- `in`
- `not-in`
- `exists`
- `not-exists`

## Rule example

```javascript
{
  field: 'employmentDetails.isDirector',
  operator: 'equals',
  value: true
}
```

The rule is declared in field metadata. React components do not contain the
business condition.

## Behaviour

The visibility engine:

- evaluates nested Staffology-native employee values;
- supports all-rules and any-rule modes;
- prevents hidden fields from being rendered;
- creates an immutable visible workspace projection;
- excludes hidden required fields from readiness calculations;
- supports future section-level visibility rules.

## Readiness integration

A conditionally hidden field is not considered required while its condition
is false. When the condition becomes true, the field immediately participates
in Creation, Payroll or RTI readiness according to its `requiredFor`
metadata.

## Governing rule

Visibility belongs to payroll metadata. Frontend components render the
result but must not duplicate conditional payroll rules.
