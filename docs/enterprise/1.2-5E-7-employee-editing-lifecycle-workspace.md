# Release 1.2 — 5E-7

## Enterprise Employee Editing & Lifecycle Workspace

This release extends the canonical Employee Workspace with:

- employee detail editing;
- version-aware `PUT /api/employees/:employeeId`;
- employment status changes;
- leaver and rehire actions;
- suspension and activation;
- validation before update;
- optimistic-concurrency payloads using `expectedVersion`;
- post-save workspace reload;
- audit-aware lifecycle messaging.

The frontend integrates directly with the existing Enterprise Employee API
Boundary introduced in Release 5E-4.
