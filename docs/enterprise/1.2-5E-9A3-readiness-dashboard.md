# Release 1.2 — 5E-9A3

## Canonical Payroll Readiness Dashboard

This release introduces continuous, metadata-driven employee readiness.

## Readiness levels

- Creation
- Payroll
- RTI

The readiness engine inspects each field's `requiredFor` metadata. It does
not hard-code payroll field names.

## Dashboard output

The dashboard returns:

- status by readiness level;
- completion percentage;
- required and completed totals;
- missing-field details;
- section readiness;
- calculation timestamp;
- employee version;
- workspace version;
- Staffology contract fingerprint.

## Status model

- `ready` — all required fields have values;
- `incomplete` — one or more required fields are missing;
- `blocked` — reserved for future blocking validation rules;
- `not-applicable` — no fields apply to the readiness level.

## Frontend boundary

This release returns an immutable dashboard model. React presentation is
intentionally deferred. The frontend may render cards, progress bars and
section indicators without owning the readiness rules.

## Governing rule

Readiness is derived from payroll metadata and employee values. It must not
be duplicated in frontend components or provider-specific screens.
