# Release 1.2 RC1-H4A

## Employer Runtime Foundation

H4A is the first package in the RC1-H4 frontend integration sequence.

It introduces:

- a single Enterprise runtime API client;
- employer discovery;
- employer selection context;
- persisted employer selection;
- an employer selector component;
- a runtime guard;
- initial runtime styling.

The package is intentionally additive. It does not modify the current
application route tree or Provider Centre. Those changes follow in H4C
after employee discovery is introduced in H4B.

## Runtime API

The frontend now has client functions for:

- listing employers;
- reading one employer;
- listing employer employees;
- reading one employee;
- loading an employer-scoped employee workspace.

## Selection model

The selected employer is stored in React context. Only the selected
employer identifier is persisted in browser local storage. It remains
runtime UI state and is not placed in `.env` configuration.
