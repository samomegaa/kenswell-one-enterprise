# Version 1.2 RC1-B

## Metadata-Driven Workspace Shell & Navigation

RC1-B switches the Employee Workspace from the hard-coded
`EMPLOYEE_SECTIONS` model to the Enterprise Workspace payload introduced in
RC1-A.

The React shell now consumes:

- visible workspace sections;
- canonical section ordering;
- section field counts;
- section readiness state;
- provider synchronisation state;
- canonical contract metadata.

RC1-B deliberately renders a section summary rather than individual field
controls. Dynamic controls are introduced in RC1-C.

The existing `WorkspaceLayout`, `WorkspaceHeader`, `WorkspaceNavigation`,
`WorkspaceSkeleton` and `WorkspaceError` components remain in use.
