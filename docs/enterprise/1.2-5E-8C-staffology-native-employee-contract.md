# Release 1.2 — 5E-8C

## Staffology Native Employee Contract

The complete Staffology Employee API payload is adopted as the first authoritative Kenswell payroll employee contract.

The system preserves every field returned by Staffology. Differences between payroll products are treated mainly as presentation, tab classification, naming, enumerations and transport differences.

Core rules:

1. Preserve the complete Staffology employee payload.
2. Do not discard unknown or newly introduced fields.
3. UI tabs reorganise fields without changing the contract.
4. Mandatory validation is separate from field existence.
5. Round-trip updates preserve untouched fields.
6. A structural fingerprint detects provider contract changes.

Run:

```bash
node tools/assimilate-staffology-employee-contract.js
```

Generated evidence is written under `review/1.2/PAYROLL/staffology/native-contract/` and must remain outside source control because it may contain personal information.
