# Release 1.2 — 5E-9A2

## Canonical Field Registry & Dynamic Renderer

This release introduces the metadata layer that describes payroll fields
and renders them through registered controls.

## Capabilities

- immutable field definitions;
- provider-path bindings;
- section assignment;
- type and control catalogues;
- readiness metadata;
- validator metadata;
- conditional-visibility metadata;
- nested value reading and immutable updates;
- pluggable control renderers;
- workspace definitions containing their fields.

## Rendering boundary

The enterprise layer produces renderer-neutral control descriptors such as:

```json
{
  "component": "TextField",
  "id": "personalDetails.firstName",
  "label": "First name",
  "value": "Amina"
}
```

React components will consume those descriptors in a later frontend
integration release. This keeps the enterprise payroll package independent
of React and browser concerns.

## Initial metadata

The release includes an initial set of high-value Staffology-native fields.
The registry is intentionally extensible. The assimilated contract can be
used to add every remaining Staffology field without modifying the dynamic
renderer.

## Governing rule

Payroll fields belong to metadata. User-interface components render the
metadata and must not become the authoritative source of payroll structure.
