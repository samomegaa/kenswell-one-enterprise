# Release 1.0-B13 — Enterprise Idempotency & Retry Boundary

## Status

Completed.

## Purpose

This release introduces the Enterprise Idempotency and Retry Boundary for Kenswell One Enterprise 1.0.

It prevents duplicate execution of state-changing operations and provides a retry contract for transient failures.

## Idempotency Model

The idempotency model introduces:

- `MemoryIdempotencyStore`
- `IdempotencyBoundary`
- `createRequestHash`

Completed idempotent operations return the original result when replayed with the same key and input.

If the same key is reused with different input, the boundary rejects the request.

## Retry Model

The retry model introduces:

- `RetryPolicy`
- `RetryBoundary`

Retries are controlled by:

- maximum attempts
- delay
- retryable predicate
