# Release 1.2 RC1-H4C

## Provider Centre Runtime Integration

This reworked H4C release preserves the established Provider Centre
interface and integrates the H4A/H4B runtime as an additive sidecar.

The existing interface retains:

- Staffology provider presentation;
- the five employer cards;
- employer-to-client linking;
- linked-client information;
- canonical employee counts;
- the existing `Open client payroll` action.

The new integration adds an expandable Enterprise runtime section
beneath the current Staffology workspace.

The sidecar supports:

- dynamic employer selection;
- employer-scoped employee discovery;
- employee search;
- empty employee states;
- employer-scoped workspace loading.

`ProviderCentre.jsx` is patched only at the component boundary:
`StaffologyProviderWorkspace` is replaced by
`IntegratedStaffologyWorkspace`. The original Staffology component is
still rendered inside that wrapper.

This avoids a wholesale Provider Centre rewrite and keeps the tested
linked-client workflow intact.
