# Release 1.2 RC1-H4B

## Employee Discovery & Workspace Launcher

H4B builds on the employer runtime introduced in H4A.

It adds:

- employer-scoped employee discovery;
- employee runtime context;
- employee selection persistence by employer;
- employee search and selection;
- runtime guards;
- employer-scoped employee workspace loading;
- a workspace launcher.

## Employee route

Employees are loaded from:

```text
GET /api/employers/:employerId/employees
```

## Workspace route

Workspaces are loaded from:

```text
GET /api/employers/:employerId/employees/:employeeId/workspace
```

## Runtime workspace contract

The frontend workspace runtime resolves:

```js
{
  employer,
  employee,
  workspace
}
```

This removes the former employee-only workspace assumption.

## Persistence

Only UI selections are stored in browser local storage. Employee
selection is stored separately for each employer. No runtime
identifier is written into environment configuration.

## Integration boundary

H4B remains additive. H4C will mount the combined
`EnterpriseRuntimeProvider`, integrate Provider Centre and introduce
the employer-scoped route tree.
