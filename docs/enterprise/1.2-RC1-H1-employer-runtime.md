# Release 1.2 RC1-H1

## Enterprise Employer Discovery & Runtime Context

RC1-H1 removes employer and employee identifiers from environment configuration and introduces dynamic Staffology employer discovery.

### Runtime model

1. Staffology credentials authenticate the provider.
2. Employers are discovered at runtime.
3. The selected employer becomes request context.
4. Employees are retrieved within that employer context.
5. Employee workspaces are created within the same context.

### API shape

- `GET /api/employers`
- `GET /api/employers/:employerId`
- `GET /api/employers/:employerId/employees`
- `GET /api/employers/:employerId/employees/:employeeId`
- `GET /api/employers/:employerId/employees/:employeeId/workspace`

Employer and employee identifiers are operational data. They are never written to `.env.local`.

### Promotion strategy

H1 is initially installed beside H0. Live discovery must confirm the Staffology response shape before the H0 production composition is replaced. This keeps the working H0 health endpoint available while the multi-employer runtime is validated.
